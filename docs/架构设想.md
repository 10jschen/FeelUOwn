# 架构设想
## 架构设想 0.1

因为主界面分为两大部分：上部分和下部分。上部分又包含用户信息区域和结果显示区域。

**视图**
从这个角度来看，分为三个widget

**控制**
暂时把这部分绑定在相应的widget类中

**数据**
基本依据api来确定各种传输数据

**缺点(待解决问题)**
1. 一个子widget发送的信号如果太多，容易混乱
2. 几个widget实际都有很多相同的函数有着大概相同的作用，不知道是否能优化
3. 分成三个widget类，实际上以后增多组件的话，会比较难维护
